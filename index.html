Comparativa de Layout

Layout

NOTA: Al hablar de Layout, nos enfocaremos más que nada en la estructura/orden/disposición visual de los elementos de una Web, además de la navegabilidad/operatividad/accesibilidad de la misma.

HTML5 UP: en este caso, el Layout de la Web no tiene menú vertical/horizontal debajo del encabezado. Todos los elementos (Templates) están volcados sobre un listado sencillo, sin posibilidad de filtros ni orden de elementos.

FREEHTML5.CO: en este caso, el Layout de la Web tiene un menú horizontal desplegable para filtrar sencillamente por categorías (tipos de Template).

Webflow: en este caso, el Layout de la Web tiene un menú vertical en la parte izquierda del cuerpo, con un filtro más complejo que el de los anteriores. No solo se filtra por categorías, sino por costo, y además tiene implementado un SearchBox de búsqueda.

Rendimiento

NOTA: Al hablar de Rendimiento, nos enfocaremos más que nada en Bugs (errores) en cada Web, como desajustes visuales.

HTML5 UP: en este caso, la Web es la más simple a nivel estructural; pero a simple vista, parece ser la mas estable en cuanto a bugs y errores visuales, así como en cuestiones de Responsive, sin grandes alteraciones.

FREEHTML5.CO: en este caso, la Web posee un menú horizontal desplegable, sin embargo, mantiene un esquema simple en términos visuales; aunque en temas Responsive tiene unas cuestiones de visualización de elementos.

Webflow: en este caso, la Web posee un menú vertical en la parte izquierda del cuerpo, y al tener un scroll (elemento con desplazamiento) ya se torna algo difícil en lo visual; en cuestiones Responsive tiene algunos bugs de visualización/superposición de elementos.

Conclusiones

A modo de listado de puntos importantes a tener en consideración para la comparativa final de las Webs, propondría lo siguiente...

Blogs/FAQ's/Recursos
En el caso de HTML5 UP no se visualizan Blogs/FAQ's/Recursos para desarrollo/edición de Templates.
En cambio, en FREEHTML5.CO y Webflow si se visualizan Blogs/FAQ's/Recursos como enlaces para indagar cuestiones técnicas o consultar con comunidades.

De lo ya mencionado sobre el tema del Layout, también es un factor a tener en cuenta para la comparativa de los resultados seleccionados; ya que nos interesa saber si se pueden filtrar los elementos, si tienen un SearchBox de búsqueda, entre otras cuestiones (Rendimiento/Bugs).

\8 Consejos de Diseño de Sitio Web para Pequeñas Empresas 🔗
Mailchimp | Biblioteca de marketing | Sitios web

8 Consejos de Diseño de Páginas Web para Pymes

Diseñar un sitio web eficaz para un negocio pequeño puede ser un reto. Especialmente, si acabas de empezar en el mundo del diseño web. Tanto si trabajas con un diseñador web profesional como si te preguntas cómo diseñar un sitio web por tu cuenta, hay algunos consejos clave que debes tener en cuenta sobre el proyecto de diseño de página web para tu pyme.

1. Usa una imagen de marca coherente
Tener en cuenta colores, tipografías, logotipos y otros elementos visuales en el sitio web y en los materiales de marketing físicos. Esto puede ayudarte a crear una identidad de marca cohesiva.
También considerar la creación de una guía de imagen de marca para usarla como referencia a lo largo del tiempo. Ayudará a tener un diseño coherente.

2. Piensa en los usuarios de móviles
En el contexto digital actual, es importante diseñar el sitio web pensando tanto en los usuarios de ordenadores como de móviles. Esto es, en pantallas de todos los tamaños.
Garantizar que el sitio web sea accesible para los usuarios de móviles, siguiendo las mejores prácticas de diseño específicas para los sitios web de pequeñas empresas. Algunas claves, como usar tipografías grandes y claras que sean fáciles de leer en pantallas pequeñas, utilizar un diseño sencillo que sea fácil de explorar y que no esté saturado, y optimizar las imágenes para garantizar que se carguen rápidamente y no ralenticen el sitio.

3. Optimiza el sitio para los motores de búsqueda
Es esencial que optimices el sitio web para los motores de búsqueda, como Google. Esto ayudará a garantizar que el sitio web aparezca en los resultados relevantes del motor de búsqueda cuando la gente busque servicios o productos de pequeñas empresas.
Seguir los estándares, las prácticas recomendadas e importantes de SEO, más que nada para los sitios web de pequeñas empresas.

4. Pon la información de contacto en la primera pantalla
La primera pantalla es la parte del sitio web visible para un usuario que visita por primera vez el sitio, sin desplazarse.
Por ende, la información de contacto debe de aparecer de forma destacada en el sitio, tendría que ser fácil de buscar.

5. Usa espacios en blanco
Los espacios en blanco pueden ayudar a crear un sitio web más atractivo visualmente y fácil de usar, ya que simplifican leer y explorar el contenido.

6. Ten páginas rápidas
En el mundo acelerado actual, la gente tiene poca paciencia con los sitios web que se cargan con lentitud. Si tu sitio web tarda demasiado en cargarse, quizá los usuarios lo cierren y busquen una opción más rápida.
Además, también puede afectar negativamente a la clasificación en los motores de búsqueda.

7. Incluye llamadas a la acción
Para una pequeña empresa, es importante tener un sitio web que no solo se vea bien, sino que también consiga resultados. Una de las mejores formas de hacerlo es incluir llamadas a la acción (CTA) eficaces en todo el sitio.
Las llamadas a la acción son botones o enlaces que animan a los usuarios a hacer algo específico, como suscribirse a un boletín de noticias, descargar un recurso o hacer una compra.

8. Céntrate en la experiencia del usuario
Hay que centrarse es la experiencia del usuario (UX). La experiencia del usuario indica si utilizar el sitio web es fácil y agradable. Es un factor clave para determinar si los usuarios se quedarán o no en el sitio.
Estos consejos de diseño web para pequeñas empresas pueden ayudarte a crear una experiencia de usuario positiva, como utilizar tipografías fáciles de leer, hacer diseños para usuarios de móviles e incluir contenido atractivo.
Además, asegúrate de probar periódicamente el sitio web para ver si funciona correctamente y resolver cualquier incidencia que surja. A fin de cuentas, el sitio web está diseñado para atraer a los clientes potenciales y ayudar a convertirlos en clientes fieles.
